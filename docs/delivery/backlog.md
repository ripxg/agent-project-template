# Product Backlog

All project work items (PBIs) are tracked here.

---

## Backlog

| ID | Title | Status | Description |
|----|-------|--------|-------------|
|    |       |        |             |

---

## Status Values

- **Proposed** - Idea captured, not yet approved
- **Agreed** - Approved by human, ready for work
- **InProgress** - Being worked on
- **Done** - Completed and approved

---

## Adding a PBI

When the agent creates a PRD, they add a row here:

```markdown
| 1 | User Authentication | Agreed | Allow users to sign up and log in |
```

Each PBI gets a folder: `docs/delivery/[ID]/` containing:
- `prd.md` - Requirements
- `tasks.md` - Task breakdown
